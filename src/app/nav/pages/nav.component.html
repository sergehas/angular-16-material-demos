This demo illustrate navigation components & behaviors

Overview

all GUI (display) components rely on loaded <module>-routing.module.ts. So have a look to these modules, more specifically to the data attribute of routes!

There is no extra conf : just define routes + data in <module>-routing.module.ts, then refer to them in the *-nav components. Highlighting the active route (tree or tab) is managed automatically.

All the demo pages are using tab-nav, & the main menu on the left is a app-tree-nav.

Routes

data.icon (optional)
define the icon to display on the right of the navigation item. This is a fontIcon name
data.animation (optional)
define the animation to apply when this route is activated
data.roles (optional)
define the (user) roles authorized to access the route (used by guard)

tab-nav component

Display children routes of the given @Input pathas tabs. If specified, navigate automatically to the @Input defaultchild path.

<Route>.data.icon(optional)
define the icon to display on the right of the navigation item. This is a fontIconname
<Route>.data.animation(optional)
define animation to apply when entering/leaving teh route. allowed values are(see src/app/shared/animations/route-animation.ts):
  • slideTopview slides from top when entering. Reverse when leaving
  • slideBottomview slides from bottom when entering. Reverse when leaving
  • slideLeftview slides from left when entering. Reverse when leaving
  • slideRightview slides from right when entering. Reverse when leaving
  • slideTabapplicable only to routes display as tab-nav. Slide from lef or right depending on the previous active tab. See angular component issue for details

tree-nav component

Display all routes in a tree view.

<Route>.data.icon(optional)
same as for tab-nav
<Route>.data.animation(optional)
same as for tab-nav, but slideTab is of course not usable (no tabs here).

LoginService Service

this service (to be adapted to support login/auth process) is the base service for all permission checks (directives & guards) feature.

appAnyRoles and appAllRoles directives

check & report if user has role(s)

@Input
array of required roles (any of or all of depending on the directive
@granted:boolean
true if user are required role(s)

Sample

<div [appAnyRoles]="['ADMIN', 'READER']" (granted)="myCompGranted=$event" >...

anyRoleGuard and allRoleGuard guards

route guard (CanActivateFn) using <Route>.data.roles

use them as canActivate function in your routing modules
