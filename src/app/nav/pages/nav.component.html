This demo illustrate navigation components & behaviours

Overview

all GUI (display) components rely on loaded <module>-routing.module.ts. So have a look to these modules, more specificaly to the data attribute of routes!

There is no exta conf : just define routes + data in <module>-routing.module.ts, then refer to them in the *-nav components. Higligting the active route (tree or tab) is managed automatically.

All the demo pages are using tab-nav, & the main menu on the left is a tree-nav.

Routes

data.icon (optional)
define the icon to display on the right of the navigaiton item. This is a fontIcon name
data.animation (optional)
define the animation to apply when this route is activated
data.roles (optional)
define the (user) roles authorized to access the route (used by guard)

tab-nav

Display children routes of the given @Input pathas tabs. If specfied, navigate automaticaly to the @Input defaultchild path.

<Route>data.icon(optional)
define the icon to display on the right of the navigaiton item. This is a fontIconname
<Route>data.animation(optional)
define animation to apply when entering/leaving teh route. allowed values are(see src/app/shared/animations/route-animation.ts):
  • slideLeftview slides from left when entering. Reverse when leaving
  • slideRightview slides from right when entering. Reverse when leaving
  • slideTabapplicable only to routes display as tab-nav. Slide from lef or right depending on the previous active tab. See angular component issue for details

tree-nav

Display all routes in a tree view.

<Route>data.icon(optional)
same as for tab-nav
<Route>data.animation(optional)
same as for tab-nav, but slideTab is of course not usable (no tabs here).
