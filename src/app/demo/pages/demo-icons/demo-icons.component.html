This demo illustrate iconlib service & registry

Overview

Teh goal of the iconsService is to provide a easy way to add a large set of static icons (static: part of the GUI app bundle) & use them with as a collection (in addition to be usable as regular mat-icon)

1st, the src/assets/iconlib.json should be generated at compilation time. See package.json build:iconlib. Basically, it just reflect the scanned directory folder tree.

Then, ar runtime / app startup, the iconlib il loaded by the IconsService with a factory to ensure lib is loaded before any page is displayed

finally, all the icons loaded from the iconlib are usable as regular angular icon, with namespace parameter

Column widths can be defines by css or by config
