Overview

This datasouce support (optinal) natively sorting & paging.
It implements native Angular native datasource, & so can be used in any datasource awre component (such as table)
It is design to avoid extra underlyling (http) service call to preserve response time & API cosumption plan.
key benefit: it requrire a simple service injection... nothing else!

Events

driving the datasource: it is supoer simple: the constructor takes 1 optional arg to control if the DS should automaticaly load at startup. Then, everithing else is automated: set the pager, set the sort, set the filter they will fire the proper (datasource internal) event to load and count (if needed). To enforce datasource loading ; simply set a filter : datasource.filter=({})

It emits events while starting/ending to load & count.

This demo illustrate the datasource in action. It is a simple table with sorting & paging enabled. The datasource is loaded with a real service using http service call (GitHb API on material repository issues).
